# 脑锻炼手势训练系统实现报告

> 说明：本报告依据当前工程代码整理，覆盖系统完整设计、智能体能力、前后端实现与部署方案。按照你的要求，效果图（截图）本版暂不提供，先保留占位说明。

## 1. 项目概述

本系统是一个面向认知训练场景的 Web 应用，核心目标是通过摄像头手势识别驱动小游戏交互，提升用户的专注力、反应力和工作记忆能力。

系统采用前后端分离架构：
- 前端负责页面交互、摄像头采集、训练流程与结果展示；
- 后端负责用户认证、游戏会话管理、手势识别推理、训练数据持久化与日报统计；
- 智能体能力体现在「视觉手势识别」与「难度自适应策略」两个模块。

## 2. 系统开发技术架构

### 2.1 总体架构

```text
[Browser / Frontend React]
  ├─ 页面路由与交互 (React + React Router)
  ├─ 摄像头采集 (WebRTC getUserMedia)
  ├─ 帧采样与调用后端识别API
  └─ 训练流程控制与报告展示
            |
            | HTTP/JSON (REST)
            v
[Backend FastAPI]
  ├─ Auth API (注册/登录/首测状态)
  ├─ Vision API (手势识别)
  ├─ Games API (会话创建/动作提交/交互记录/日报)
  ├─ GameEngine (游戏规则与计分)
  ├─ GestureModelService (模型推理服务)
  └─ SQLiteStore (本地数据库持久化)
            |
            v
[SQLite + 模型文件]
  ├─ backend/data/game_data.db
  ├─ GestureRecognizer/gesture_model.joblib
  └─ hand_landmarker.task
```

### 2.2 前端技术栈（Web 应用开发框架）

- 框架：`React 19` + `TypeScript` + `Vite`
- 路由：`react-router-dom`
- 核心能力：
  - 单页应用路由与鉴权拦截（登录态、首测完成态）
  - 摄像头预览与实时检测事件分发
  - 游戏页面（算式回溯、数箱子、石头剪刀布）
  - 报告页统计与 JSON 导出

### 2.3 后端技术栈（服务框架）

- 框架：`FastAPI` + `Uvicorn`
- 视觉与模型推理：`MediaPipe` + `OpenCV` + `joblib` + `scikit-learn`
- 数据存储：`SQLite`
- 数据模型：`Pydantic`
- 核心能力：
  - RESTful API 分层（`/auth`、`/vision`、`/games`）
  - 会话生命周期管理（创建、轮询、提交动作、结束）
  - 训练行为落库（会话表 + 交互表 + 用户表）
  - 日报汇总（时长、局数、各游戏统计、剩余局数）

### 2.4 智能体实现说明

本项目智能体能力由两个子模块构成：

#### A. 手势识别智能体（视觉推理）
- 输入：前端上传的相机帧 `image_base64`
- 处理链路：
  1. Base64 解码为图像；
  2. MediaPipe 检测手部关键点（21 点）；
  3. 特征归一化（以腕点平移 + 尺度归一）；
  4. `joblib` 模型执行 `predict_proba`；
  5. 置信度阈值过滤（默认 `0.30`）并输出最终手势标签。
- 输出：`gesture`、`confidence`、`hand_detected` 等字段。

#### B. 难度自适应智能体（训练策略）
- 作用对象：`算式回溯` 游戏。
- 策略：读取用户最近交互正确率，动态调整 `n_back_level`：
  - 正确率 `< 60%`：降低一级（最低 1）；
  - 正确率 `> 85%`：提高一级；
  - 中间区间：维持不变。
- 价值：让训练强度跟随用户能力变化，避免“一刀切”难度。

## 3. 服务部署方案介绍（虚拟机 + 操作系统）

### 3.1 部署方式

本项目采用前后端分离的同机部署方案，使用脚本 `deploy.sh` 一键执行：
- 前端：`npm run build` 后使用 `vite preview` 监听 `4173`；
- 后端：`uvicorn app.main:app` 监听 `8000`；
- 数据库：`backend/data/game_data.db`（自动初始化）；
- 进程管理：脚本记录 PID 到 `.run/`，便于重启与停止。

### 3.2 虚拟机租用方案（可直接用于答辩/文档）

> 当前代码在 Linux 环境可运行。以下为推荐的生产/演示虚拟机方案（若你已有具体云厂商和规格，可直接替换本节参数）。

- 云主机类型：通用型 ECS/CVM
- 推荐规格：`2 vCPU / 4 GB RAM / 50 GB SSD`
- 操作系统：`Ubuntu 22.04 LTS`（或同等 Linux 发行版）
- 网络与端口：
  - `4173/tcp`：前端访问
  - `8000/tcp`：后端 API 与 Swagger 文档
  - `22/tcp`：运维 SSH
- 运行时环境：
  - `Node.js 20+`
  - `Python 3.10+`
- 部署步骤：拉取代码后执行 `bash deploy.sh`

### 3.3 建议的生产增强（可选）

- 使用 `Nginx` 做反向代理（统一 80/443 入口）
- 使用 `systemd` 或 `supervisor` 守护 FastAPI 与前端服务
- 配置 HTTPS 证书（Let’s Encrypt）
- 按需拆分前后端主机，提升扩展性与安全性

## 6. 前后端功能实现清单（可用于“实现说明”章节）

### 6.1 前端已实现功能
- 登录/注册/角色选择/评估流程；
- 路由鉴权与首测完成态控制；
- 摄像头预览、手势事件监听、空格确认提交流程；
- 三类训练游戏的 UI/状态机/结果跳转；
- 报告页查询与 JSON 导出；
- 个人中心与本地数据管理。

### 6.2 后端已实现功能
- 认证接口：注册、登录、用户资料查询、首测状态更新；
- 视觉接口：手势识别推理（Base64 图像输入）；
- 游戏接口：会话创建、动作提交、交互查询、日报查询；
- 游戏引擎：计分逻辑、状态推进、会话完成判断；
- 数据层：SQLite 自动建表、会话/交互/用户持久化。

### 6.3 智能体已实现功能
- 手部关键点检测（MediaPipe）；
- 特征工程与分类模型推理（joblib 模型）；
- 置信度阈值控制 + 异常输入处理；
- 基于历史正确率的 N-back 难度自适应。
