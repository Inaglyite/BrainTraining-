# 脑锻炼小游戏前端原型详细设计报告

## 1. 概述与设计目标
本项目为一款以“脑部锻炼”为主题的小游戏系统原型。通过结合设备摄像头实现动作/手势识别，系统提供了轻量化、互动性强的游戏体验。
前端设计遵循**现代化、极简主义与响应式**的原则，使用 React 框架搭建，并围绕“用户信息采集 → 角色定制 → 评估与引导 → 游戏训练 → 数据报告与管理”构建了完整的闭环。

---

## 2. 应用页面组织结构

### 2.1 整体架构视图
应用采用扁平化的路由层级，通过 `React Router` 进行模块调度。各个页面的功能划分如下：

1. **入口模块**
   - **注册页面** `/`：用户初次访问，输入基础信息。
   - **角色选择** `/role-select`：完成分流，以定制化目标引导用户。
   - **评估引导** `/evaluation-choice`：询问并分发至系统引导（校准/评估）或游戏大厅。
   - **摄像头校准** `/onboarding`：环境检测与参数校验。
2. **中心大厅**
   - **开始页面（主界面）** `/start`：聚合所有核心游戏模块和管理功能的入口。
3. **游戏训练模块（统一使用两栏布局：左侧信息与监控，右侧游戏主体）**
   - **算式回溯** `/games/suan-shi`：工作记忆与计算训练。
   - **数箱子** `/games/shu-xiang`：手眼协调与计数专注力。
   - **石头剪刀布** `/games/rps`：决策判断与快速反应。
4. **用户与数据模块**
   - **锻炼报告** `/reports`：呈现数据结果及系统事件记录。
   - **个人中心** `/profile`：用户属性管理和基础系统控制（重置/清除数据）。

## 3. 详细页面交互与原型说明

### 3.1 注册页面 (Registration)
- **主要功能**：收集用户的“昵称”和“生日”信息，注册到本地存储环境。
- **页面原型**：
  - 居中的浮动卡片（卡片宽度及间距经过优化适配）；
  - 带有系统主题的图标、主视觉标题“欢迎来到脑锻炼小游戏”。
  - 表单部分包含文本输入框和日期选择器。
- **交互逻辑**：
  - 用户提交表单后，系统使用 `saveUser()` 发起本地数据写入。
  - 完成写入后，自动路由跳转至 `/role-select` 页面。

### 3.2 角色选择页面 (RoleSelect)
- **主要功能**：支持用户选择定制化角色（学生党、上班族、银发族），增强代入感。
- **页面原型**：
  - 采用流式网格，水平排列三张卡片；
  - 每张卡片包含醒目的 Emoji 大图标及针对性的描述文案（如：活跃思维 / 增强记忆 / 保持大脑活力）。
- **交互逻辑**：
  - 鼠标悬停卡片会触发上浮（`translateY`）及扩散阴影的动画效果。
  - 点击卡片即可触发存储事件，将 `role` 属性写入本地后即刻跳转至 `/evaluation-choice`。

### 3.3 评估引导页面 (EvaluationChoice)
- **主要功能**：建立“基准评估”概念，用于询问用户是否需要参与初始化测评。
- **页面原型**：
  - 居中的说明性卡片，带有显眼的指引图标及友好的描述文本。
  - 底部平行放置“跳过，直接开始”（低优按钮）与“开始评估”（高优主色按钮）。
- **交互逻辑**：
  - 若点击“跳过”，则直接路由到 `/start` 进入系统默认状态。
  - 若点击“开始评估”，则路由到 `/games/suan-shi` 直接开启第一个强制测试模块。

### 3.4 游戏中心页面 (Start)
- **主要功能**：向用户展示系统提供的全部训练应用及用户数据中心通道。
- **页面原型**：
  - 采用现代化设计语言的看板布局。
  - 游戏入口使用统一结构的深景深卡片显示（各自配备图标和简要说明）。
  - 下方使用特殊颜色主题（紫色、绿色）单独区分出“锻炼报告”与“个人中心”两大管理模块。
- **交互逻辑**：
  - 带有过渡动画，页面加载时内容标题拥有缓入上浮效果（`fadeInUp`）。
  - 连接对应的独立应用路由单元。

### 3.5 核心游戏模块布局体系
三大游戏模块（算式回溯、数箱子、石头剪刀布）针对原型进行了重构，采用了**统一的双栏布局响应式标准**：

- **左侧信息与监控区 (`left-info-panel`)**：
  - 固定宽排版。
  - 使用两张独立的高亮数据卡片分别显示 **“倒计时”** 与 **“分数”**。
  - 最底部集成无框极简形式的摄像头预览窗口（配合环境监测模块），实时反馈动向，不占用主视觉。
  
- **右侧游戏主视区 (`main-game-area`)**：
  - 占据剩余空间的弹性盒子，并带有渐变背景；中心化突出各不同游戏的业务组件。

#### (A) 算式回溯页面 (SuanShiHuiSuo)
- **主要功能**：通过心算挑战玩家记忆力；根据动作切换题目。
- **组件规划**：主体为大型强调文字样式展示当前算式（带有呼吸态脉冲动画）。
- **交互机制**：通过外部识别（手势）驱动状态。握拳（扣分），张手（提交与推进新题目），响应快速无阻。

#### (B) 数箱子页面 (ShuXiangZi)
- **主要功能**：图形化数量构建与心智映射；
- **组件规划**：基于 `Flex Wrap` 网格布局动态渲染箱子图片（采用高质量 `Box.png` 素材）。
- **交互机制**：触发增加动作后实时在网格中推入新的图片占位（同时累计分数和箱子数量），触发缩减动作时移出实例，支持自然过渡效果。

#### (C) 石头剪刀布页面 (RPS)
- **主要功能**：模拟对抗反馈，测试反应力。
- **组件规划**：建立对称的对抗舞台，两侧分别加载带有悬浮交互和镜像处理的图片库（石头、剪刀、布），中央加载结果评价字幕。
- **交互机制**：监听用户的动作并映射为系统识别的角色状态，程序随即随机生成电脑手势进行比对，胜负平逻辑处理之后直观反馈到组件面板上。

### 3.6 锻炼报告页面 (Reports)
- **主要功能**：展示操作的历史会话流供追溯。
- **页面原型**：列表形式排列的数据展示版；集成数据总结头部标签。
- **交互逻辑**：提供清除本地全部游戏记录功能，及序列化导出为 JSON 代码以便上卷分析的功能。

### 3.7 个人中心页面 (Profile)
- **主要功能**：系统配置与使用者基础信息。
- **页面原型**：
  - 统一风格结构的圆角大卡片；
  - 头部拥有字母头像和角色展示 Badge 工具（如显示：角色：上班族）。
  - 下方为一排全宽的操作入口按钮。
- **交互逻辑**：
  - 预留了“修改信息 / 修改密码”的开发弹窗。
  - 集成了重要的“重新校准摄像头”导航，及一键“退出登录并清除数据”（将毁坏用户空间本地 Key）的高危操作。

---

## 4. 总结
本应用的前端设计采用标准的现代组件化架构。通过逻辑链路分离（收集-导航-评估）、视觉体验增强（动画与阴影重构）以及操作规范约束（统一的游戏布局逻辑、免干预型的极简摄像头组件），达成了一个逻辑结构清晰、交互体验自然流畅的脑部锻炼游戏原型。能够优秀地向上层交接测试并展示智能体结合前端业务的实现效果。
